Mybatis 공식 사이트

MyBatis – 마이바티스 3 | 소개
마이바티스는 개발자가 지정한 SQL, 저장프로시저 그리고 몇가지 고급 매핑을 지원하는 퍼시스턴스 프레임워크이다. 마이바티스는 JDBC로 처리하는 상당부분의 코드와 파라미터 설정및 결과 매핑을 대신해준다. 마이바티스는 데이터베이스 레코드에 원시타입과 Map 인터페이스 그리고 자바 POJO 를 설정해서 매핑하기 위해 XML과 애노테이션을 사용할 수 있다.
https://mybatis.org/mybatis-3/ko/index.html

퍼시스턴스 프레임워크

퍼시스턴스 프레임워크는 데이터베이스와의 상호작용을 단순화하고 추상화하는 소프트웨어 도구입니다. 이 도구를 사용하면 개발자는 SQL 쿼리를 직접 작성하는 대신 객체 지향적인 방식으로 데이터베이스에 접근할 수 있습니다. Mybatis는 개발자가 지정한 SQL, 저장프로시저, 그리고 몇 가지 고급 매핑을 지원하는 퍼시스턴스 프레임워크입니다.

SQL Mapper

Mybatis는 SQL Mapper 형태로 동작합니다. 개발자는 SQL 문장이나 저장프로시저를 직접 작성하고, Mybatis는 이를 데이터베이스에 전달하고 결과를 매핑하여 반환합니다. 이러한 방식을 통해 개발자는 직접적인 SQL 작성에 대한 유연성과 제어력을 가지면서도, 객체 지향적인 방식으로 데이터베이스와 상호작용할 수 있습니다.

Mybatis vs JPA

Mybatis와 JPA는 모두 퍼시스턴스 프레임워크이지만, 다른 접근 방식과 철학을 가지고 있습니다. JPA는 객체-관계 매핑을 중심으로 한 ORM(Object-Relational Mapping) 프레임워크이며, 개발자가 객체를 통해 데이터베이스와 상호작용합니다. 반면 Mybatis는 SQL Mapper로서 직접적인 SQL 작성과 매핑을 중심으로 한 프레임워크입니다. 어떤 프레임워크를 선택할지는 개발자의 선호도와 프로젝트 요구사항에 따라 결정됩니다.

Mybatis SQL

select 문

MyBatis의 SELECT 문은 데이터베이스에서 데이터를 검색하는 데 사용됩니다. 지정된 조건에 기반하여 단일 또는 다중 행을 검색하는 데 사용할 수 있습니다. SELECT 문의 결과는 Java 객체 또는 Java 객체 컬렉션에 매핑될 수 있습니다.
<select id="select" resultType="com.joeun.test.dto.Board"> SELECT * FROM board WHERE no = #{no} </select>
XML
복사
위의 코드에서 select 문은 no 매개 변수를 기반으로 board 테이블에서 단일 행을 검색합니다. 결과는 Board 클래스에 매핑됩니다.

insert 문

MyBatis의 INSERT 문은 데이터베이스에 새로운 레코드를 삽입하는 데 사용됩니다. 단일 행 또는 한 번에 여러 행을 삽입하는 데 사용할 수 있습니다. 삽입할 값을 매개 변수로 제공하거나 Java 객체로부터 매핑할 수 있습니다.
<insert id="insert"> INSERT INTO board (title, writer, content) VALUES (#{title}, #{writer}, #{content}) </insert>
XML
복사
위의 코드에서 insert 문은 title, writer, 및 content 매개 변수에서 제공된 값으로 board 테이블에 새로운 행을 삽입합니다.

update 문

MyBatis의 UPDATE 문은 데이터베이스의 기존 레코드를 수정하는 데 사용됩니다. 지정된 조건에 기반하여 하나 이상의 열을 수정하는 데 사용할 수 있습니다. 수정할 값을 매개 변수로 제공하거나 Java 객체로부터 매핑할 수 있습니다.
<update id="update"> UPDATE board SET title = #{title}, writer = #{writer}, content = #{content}, upd_date = now() WHERE no = #{no} </update>
XML
복사
위의 코드에서 update 문은 no 매개 변수를 기반으로 board 테이블의 title, writer, content, 및 upd_date 열을 수정합니다.

delete 문

MyBatis의 DELETE 문은 데이터베이스에서 기존 레코드를 제거하는 데 사용됩니다. 지정된 조건에 기반하여 하나 이상의 행을 삭제하는 데 사용할 수 있습니다.
<delete id="delete"> DELETE FROM board WHERE no = #{no} </delete>

selectKey 문

MyBatis의 SELECTKEY 문은 INSERT 문에서 생성된 키 값을 검색하는 데 사용됩니다. 주로 데이터베이스에 새 레코드를 삽입한 후 자동으로 생성된 기본 키 값을 검색하는 데 사용됩니다.
<insert id="insert"> <selectKey resultType="java.lang.Integer" keyProperty="parentNo" order="BEFORE"> SELECT MAX( ${pk} ) FROM ${parentTable} </selectKey> INSERT INTO file (parent_table, parent_no, file_name, file_path) VALUES (#{parentTable}, #{parentNo}, #{fileName}, #{filePath}) </insert>
XML
복사
위의 코드에서 selectKey 문은 마지막으로 삽입된 ID 값을 검색하고 Java 객체의 no 속성에 매핑합니다. 이를 통해 새 레코드를 삽입한 후 생성된 키 값을 액세스할 수 있습니다.

${ } vs #{ }

•
${ } : 넘어온 매개변수의 값을, 그대로 출력
•
#{ } : 넘어온 매개변수의 값을, 자동 타입 변환하여 출력

<select id="callFunction" resultType="java.lang.Integer" statementType="CALLABLE"> { #{result, mode=OUT, jdbcType=INTEGER} = call functionName(#{param1, jdbcType=INTEGER}, #{param2, jdbcType=INTEGER}) } </select>

복사

set 문

동적 SQL을 작성하기 위해 MyBatis에서 제공하는 set 문법을 사용할 수 있습니다. set 문법은 UPDATE 쿼리에서 필요한 필드만 동적으로 업데이트할 수 있는 유용한 기능입니다. 다음은 set 문법의 예시입니다.
<update id="update"> UPDATE board <set> <if test="title != null"> title = #{title}, </if> <if test="writer != null"> writer = #{writer}, </if> <if test="content != null"> content = #{content}, </if> </set> WHERE no = #{no} </update>
XML
복사
위의 예시에서는 title, writer, content 필드가 null이 아닌 경우에만 해당 필드를 업데이트하도록 설정되어 있습니다. 이를 통해 필요한 필드만 동적으로 업데이트할 수 있습니다.
